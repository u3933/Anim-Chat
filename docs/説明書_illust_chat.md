## フォルダー構造

```
svg_anim_test/
├── illust_chat.html          # 静止画版チャットアプリ本体
├── wizard.html               # セットアップウィザード
├── start.bat                 # Windows 起動スクリプト（ダブルクリック可）
├── start.ps1                 # Windows PowerShell 起動スクリプト
├── start.sh                  # macOS / Linux 起動スクリプト
├── manifest.json             # PWA マニフェスト
├── tts_dict.yaml             # TTS 読み替え辞書（固有名詞など）
├── images/                   # サンプル表情画像（初期登録用）
│   ├── normal.png            # スロット0: ノーマル
│   ├── happy.png             # スロット1: 嬉しい
│   ├── sad.png               # スロット2: 悲しい
│   ├── surprised.png         # スロット3: 驚く
│   ├── shy.png               # スロット4: 照れる
│   ├── thinking.png          # スロット5: 考えている
│   ├── angry.png             # スロット6: 怒っている
│   └── sleepy.png            # スロット7: 眠い
├── knowledge/                # ナレッジベース（.txt / .md ファイルを配置）
├── knowledge_manifest.json   # ナレッジ読込リスト（自動生成）
└── docs/                     # ドキュメント類
```

> `images/` のサンプル画像は初期登録の参考用です。設定 → 表情設定から任意の画像に差し替えられます。

## 必要環境

| 項目 | 要件 |
|---|---|
| Node.js | v18 以上（`npx serve` のため） |
| ブラウザ | Chrome / Edge 最新版推奨（Safari は部分的に動作） |
| LLM API | Google Gemini API キー **または** OpenAI 互換エンドポイント |
| TTS（任意） | Style-Bert-VITS2 または VOICEVOX をローカルで起動済み |
| 表情画像 | キャラクターの差分イラスト（PNG / JPG 推奨） |

## セットアップ・起動方法

### Windows

1. **Node.js** をインストール（未インストールの場合）
2. `start.bat` をダブルクリック
   - ローカルHTTPサーバー（ポート 3000）を起動
   - ブラウザが `http://localhost:3000/wizard.html` を自動で開く
3. ウィザードで「起動モード」を **静止画版** に切り替えて「チャット画面を開く →」

または PowerShell から:
```powershell
.\start.ps1
```

### macOS / Linux

```bash
chmod +x start.sh
./start.sh
```

### 手動起動

```bash
npx serve . -p 3000
# ブラウザで http://localhost:3000/wizard.html を開く
```

ウィザードを経由せず直接開く場合（設定済みの場合）:
```
http://localhost:3000/illust_chat.html
```

> **重要**: `file://` プロトコルでは IndexedDB（表情画像の保存）が動作しません。必ずローカルHTTPサーバー経由で開いてください。

## 初回設定（ウィザード）

`wizard.html` を開くと以下のセクションが表示されます。

### 1. LLM（AI）設定

| プロバイダー | 設定内容 |
|---|---|
| **Gemini（Google）** | Gemini API キー + モデル名（例: `gemini-2.0-flash`） |
| **OpenAI 互換** | ベースURL（例: `http://localhost:11434/v1`）+ APIキー（ローカルは空欄可）+ モデル名 |

> **Gemini thinking 系モデル注意**: `gemini-3.1-flash-lite-preview` など thinking モデルを使用する場合、表情選択API は `maxOutputTokens: 256` を使用しています。モデルの思考トークンが多い場合はさらに値を増やす必要があります。

### 2. 音声合成（TTS）設定

| プロバイダー | 設定内容 |
|---|---|
| **Style-Bert-VITS2** | エンドポイント（デフォルト: `http://127.0.0.1:5000`）+ モデルフォルダ名 |
| **VOICEVOX** | エンドポイント（デフォルト: `http://127.0.0.1:5000`）+ スピーカーID |
| **TTS なし** | 音声なしでテキストのみ動作 |

「▶ テスト再生」ボタンで接続と音声を確認できます。

### 3. キャラクター設定

- **アプリタイトル**: ブラウザタブとヘッダーに表示
- **システムプロンプト**: AIキャラクターの人格・設定（空欄でデフォルト使用）

### 4. 外観設定

ダーク系・ライト系のカラーテーマを選択。

### 起動モード選択

- **Live2D版** → `live2d_chat.html` が開く
- **静止画版** ← このファイル

「チャット画面を開く →」で設定を保存して起動します。

## 操作説明

### 基本チャット

| 操作 | 内容 |
|---|---|
| テキスト入力 → Enter / 送信ボタン | AIにメッセージを送信 |
| ■ 中断ボタン | 生成中のAI返答を中断 |
| Shift+Enter | 改行（送信しない） |

AIの返答テキストの感情を解析し、対応する表情画像に自動で切り替わります。

### レイアウト切り替え

ヘッダーの「モバイル / PC表示」ボタンでレイアウトを切り替えます。

#### PCモード
- 左: アバター画像パネル
- 右: チャットパネル
- 2カラムレイアウト

#### モバイルモード（ビジュアルノベル風）
- アバター画像が全画面背景表示
- チャット・入力欄が半透明オーバーレイで下部に配置
- スマートフォン縦画面に最適化（幅 640px 以下で自動適用）

### 設定パネル（設定 ▼）

ヘッダー下の「設定 ▼」ボタンで展開します。

- **TTS ON/OFF**: 音声合成の有効・無効切り替え
- **Grounding ON/OFF**: Gemini の Google 検索グラウンディング（Gemini のみ）
- **背景・前景画像**: 任意の画像を IndexedDB に保存して表示

### メール機能（📪 メール）

AIが会話の流れで生成するショートメール。受信時はアイコンが 📬 に変化。

### メモリ機能

会話から自動的にユーザーの情報・好みを抽出・保存。Gist同期でクロスデバイス共有が可能。

### モノローグ機能

一定時間操作がないとき（デフォルト5分）、キャラクターが自動で独り言を発し、感情に合わせた表情に切り替わります。

### Gist 同期（☁️）

GitHub Personal Access Token と Gist ID を使って、会話履歴・メモリ・設定を別デバイスと共有できます。詳細は [gist_sync.md](gist_sync.md) を参照。

## 表情スロット設定

静止画版の最大の特徴です。10スロットにキャラクターの差分イラストを登録し、AIが会話の感情に合わせて自動選択します。

### 設定の開き方

設定パネル（設定 ▼）の中にある「表情設定」ボタンをクリック。

### デフォルトスロット構成

| スロット | ラベル | 対応感情 |
|---|---|---|
| 0 | ノーマル（通常） | 中立・日常会話 |
| 1 | 嬉しい・楽しい | 喜び・興奮・ポジティブ |
| 2 | 悲しい・寂しい | 悲哀・落ち込み |
| 3 | 驚く | 驚き・衝撃 |
| 4 | 照れる・恥ずかしい | 恥じらい・ドキドキ |
| 5 | 考えている | 思案・疑問 |
| 6 | 怒っている・不機嫌 | 不満・叱責 |
| 7 | 眠い・疲れている | 倦怠・眠気 |
| 8 | （未設定） | 任意でカスタマイズ可 |
| 9 | （未設定） | 任意でカスタマイズ可 |

### 画像の登録手順

1. 「表情設定」ボタンをクリック → 表情スロット設定モーダルが開く
2. 登録したいスロットの「画像選択」ボタンをクリック
3. 差分イラスト画像ファイルを選択（PNG / JPG 推奨）
4. サムネイルに画像が表示されれば登録完了
5. 「テスト」ボタンでアバターへの反映を確認
6. 「閉じる」で設定を保存

### 画像の保存先

画像は **IndexedDB**（ブラウザのローカルストレージ）に保存されます。ページをリロードしても保持されます。

> **注意**: ブラウザのサイトデータをクリアすると画像が削除されます。定期的にセッション保存（JSON エクスポート）でバックアップすることを推奨します（ただし Gist 同期には画像は含まれません）。

### ラベルのカスタマイズ

テキストボックスのラベルを編集することで、AIへの表情選択プロンプトを変更できます。例:
- `照れる・恥ずかしい` → `照れる・恥ずかしい・ときめき` のように感情を追加

### 表情選択の仕組み

AIが返答を生成した後、以下のプロンプトを別途 LLM に送信して表情番号を取得します:

```
You are selecting a facial expression for an anime character based on their response.
Available expressions:
0: ノーマル（通常）
1: 嬉しい・楽しい
...

User: "..."
Character's response: "..."

Reply with ONLY the single digit number of the most fitting expression.
```

- **対象**: 画像が登録済みのスロットのみ（未登録スロットは選択肢に出ない）
- **フォールバック**: LLM が応答しない・エラーの場合はスロット 0 に戻る
- **フェード効果**: 画像切替時に 0.25秒のフェードイン

## アーキテクチャ

### 全体構成

```
ブラウザ（単一HTMLファイル）
├── アバター表示（<img> 要素）
│     └── 表情切替: img.src = blobURL + opacity フェード
├── LLM API（Gemini / OpenAI互換）
│     ├── チャット返答
│     ├── 表情スロット選択（別 API 呼び出し）
│     ├── モノローグ生成
│     ├── メモリ抽出
│     └── メール生成
├── TTS API（SBV2 / VOICEVOX）
│     └── 音声 → AudioContext で再生（口パクなし）
├── localStorage
│     └── 設定・会話履歴・レイアウト・テーマ・スロットラベル
├── IndexedDB
│     ├── 表情画像 Blob（hiyori_expr_db / images ストア）
│     ├── 背景・前景画像 Blob
│     └── メモリ DB
└── GitHub Gist API（任意）
      └── セッション同期（画像を除く）
```

### チャット → 表情切替フロー

```
sendMessage()
  ↓
callLLMChat(userMsg)             ← チャット返答（メインLLM）
  ↓
[返答テキスト表示]
  ↓
callLLMExpression(userMsg, aiText)  ← 表情番号を選択（サブLLM呼び出し）
  ↓
switchExpression(id)             ← img.src を差し替え + フェード
  ↓
playTTSWithLipSync(aiText)       ← TTS 再生
```

### 表情データ管理

| データ | 保存先 | キー |
|---|---|---|
| スロットラベル | localStorage | `hiyori_expr_slots` |
| 画像 Blob | IndexedDB | `hiyori_expr_db` / `images` ストア（キー = スロットID） |
| 実行時 URL | メモリ（変数） | `expressions[i].image`（blob URL） |

### モバイルレイアウトの z-index 構成

```
z-index: 30  ── app-header（固定ヘッダー）
z-index: 25  ── メールボタン / 設定ボタン
z-index: 24  ── 設定パネル
z-index: 10  ── chat-panel（半透明オーバーレイ）
z-index:  0  ── avatar-panel（全画面背景）
```

## トラブルシューティング

### 表情が切り替わらない

**原因1: 画像が登録されていない**
- 「設定 → 表情設定」でスロットにサムネイルが表示されているか確認
- 画像未登録のスロットは選択肢に含まれないためスロット 0 のままになる

**原因2: LLM の応答トークンが不足**
- Gemini thinking 系モデル（`gemini-x.x-flash-lite-preview` など）では、思考トークンが `maxOutputTokens` を消費して出力が空になる
- 症状: コンソールに `finishReason: "MAX_TOKENS"` かつ `content: {}` が表示される
- 対処: `callLLMExpression` 内の `maxOutputTokens` を `256` 以上に設定（現在のデフォルト値）

**原因3: LLM API エラー**
- コンソールで `[Expr LLM] HTTP 4xx` を確認
- API キー・エンドポイントが正しいか確認

### 画像がリロード後に消える

- ブラウザのサイトデータ・キャッシュをクリアした可能性あり
- プライベートブラウズモードでは IndexedDB がセッション終了時に削除される
- 通常のウィンドウで使用してください

### モバイルモードでボタンが見えない

- ブラウザのズームレベルを確認（100% 推奨）
- ヘッダー下に「メールボタン」「設定ボタン」が縦に並んでいるか確認
- キャッシュが古い場合は強制リロード（Ctrl+Shift+R）

### AIが返答しない

- ウィザードで API キーが保存されているか確認
  - DevTools → Application → localStorage → `hiyori_cfg_gemini_api_key`
- コンソールに `[Chat]` エラーが出ていれば内容を確認
- Gemini の場合: モデル名が正しいか確認（例: `gemini-2.0-flash`）
- OpenAI互換の場合: エンドポイントが起動中か確認

### TTS が再生されない

- TTS サーバーが起動しているか確認
- ウィザードの「▶ テスト再生」でエラーを確認
- 設定パネルの「TTS ON」になっているか確認
- Style-Bert-VITS2 の場合: モデルフォルダ名がサーバー側と一致しているか確認

### 設定がリセットされる

- localStorage がブロックされている（プライベートブラウズ・拡張機能の干渉）
- ポート番号が変わった場合は別オリジンとして扱われ、localStorageが共有されない
